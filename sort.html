SORT

El comando SORT se utiliza para ordenar el texto de un archivo.

Parámetros:

-b, --ignore-leading-blanks

Ignora los espacios en blanco al principio de cada línea

-d, --dictionary-order

Considera solamente caracteres alfanuméricos y espacios en blanco

-f, --ignore-case

Ignora las diferencias entre mayúsculas y minúsculas al comparar

-g, --general-numeric-sort

Ordena según el valor numérico general

-i, --ignore-nonprinting

Ignora los caracteres no imprimibles al comparar

-M, --month-sort

Ordena por nombres de meses: 'ENE' < ... < 'DIC'

-h, --human-numeric-sort

Ordena números legibles para humanos (por ejemplo, 2K 1G)

-n, --numeric-sort

Ordena según el valor numérico de la cadena

-R, --random-sort

Baraja las líneas, pero agrupa claves idénticas

--random-source=FILE

Utiliza FILE como fuente de datos aleatorios

-r, --reverse

Invierte el resultado de las comparaciones

--sort=WORD

Ordena según WORD: numérico general -g, numérico humano -h, mes -M, numérico -n, aleatorio -R, versión -V

-V, --version-sort

Ordenación natural de números de versión dentro del texto

Otros parámetros:

--batch-size=NMERGE

Fusiona como máximo NMERGE entradas a la vez; para más, use archivos temporales

-c, --check, --check=diagnose-first

Verifica si la entrada está ordenada; no realiza la ordenación

-C, --check=quiet, --check=silent

Como -c, pero no informa sobre la primera línea incorrecta

--compress-program=PROG

Utiliza PROG para comprimir temporales; descomprímelos con PROG -d

--debug

Anota la parte de la línea utilizada para ordenar y advierte sobre el uso cuestionable a stderr

--files0-from=F

Lee la entrada de los archivos especificados por nombres terminados en NUL en el archivo F; Si F es -, lee los nombres desde la entrada estándar

-k, --key=KEYDEF

Ordena mediante una clave; KEYDEF da ubicación y tipo

-m, --merge

Fusiona archivos ya ordenados; no realiza la ordenación

-o, --output=FILE

Escribe el resultado en FILE en lugar de la salida estándar

-s, --stable

Estabiliza la ordenación deshabilitando la comparación de último recurso

-S, --buffer-size=SIZE

Utiliza SIZE para el búfer de memoria principal

-t, --field-separator=SEP

Utiliza SEP en lugar de la transición de no blanco a blanco como separador de campos

-T, --temporary-directory=DIR

Utiliza DIR para temporales, no $TMPDIR o /tmp; varias opciones especifican varios directorios

--parallel=N

Cambia el número de ordenaciones ejecutadas concurrentemente a N

-u, --unique

Con -c, verifica el orden estricto; sin -c, muestra solo el primero de una ejecución igual

-z, --zero-terminated

El delimitador de línea es NUL, no una nueva línea

--help

Muestra esta ayuda y sale

--version

Muestra información de versión y sale

Ejemplos:

Mostrar archivo ordenado alfabéticamente (a-z).

sort myfile.txt

-!?

012345

abcdefg

cdefg

uvwxyz

xyz

Mostrar archivo ordenado alfabéticamente en reversa (z-a).

sort myfile.txt -r

xyz

uvwxyz

cdefg

abcdefg

012345

-!?

Mostrar archivo ordenado numéricamente (ascendente).

sort numbers.txt -n

10

20

30

40

50

Mostrar archivo ordenado numéricamente (descendente).

sort numbers.txt -n -r

50

40

30

20

10

Mostrar archivo ordenado alfabéticamente por nombre de mes.

sort dates.txt -M

Enero

Febrero

Marzo

Abril

Mayo

Junio

Mostrar archivo ordenado alfabéticamente ignorando mayúsculas y minúsculas.

sort text.txt -f

apple

Banana

banana

cherry

Cherry

Mostrar archivo ordenado alfabéticamente considerando solo caracteres alfanuméricos y espacios en blanco.

sort alpha_numeric.txt -d

456 bananas

Bananas 456

789 cherries

Cherries 789

sort random.txt -R
